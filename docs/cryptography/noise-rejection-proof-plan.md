# Noise-Bound and Rejection-Sampling Proof Plan

Date: 2026-05-27

## Scope

This document lists the proof obligations needed before the threshold
ML-DSA-65 scaffold can claim standard ML-DSA compatibility or production
security. It complements `correctness-lemmas.md` by focusing on infinity-norm
bounds, Fiat-Shamir-with-aborts behavior, and rejection-sampling distribution
preservation.

The current repository contains only simulation and arithmetic scaffolding. The
real ML-DSA-65 backend, bridge test, noise-bound proof outline, and protocol
spec requested for review were not present in this checkout. All real-backend
claims below are obligations, not implemented facts.

## Objects to Specify

The complete proof must first define these objects for ML-DSA-65:

- Ring and module domains used by the backend, including coefficient/NTT
  conversion points.
- Public key `pk`, secret shares `s_i`, aggregate secret `s`, message `m`, and
  transcript challenge `c`.
- Local masking shares `y_i`, commitments to masking material, aggregate mask
  `y`, partial responses `z_i`, aggregate response `z`, and hint vector `h`.
- Active signer set `A` and Lagrange coefficients `lambda_i(A)`.
- Norm bounds for `z`, low bits, hints, and challenge weight for the selected
  ML-DSA-65 parameter set.
- Abort predicate `Reject(...)` and retry/attempt transcript binding.

## Proof Goal

For any execution that outputs an aggregate signature `sigma`, prove:

```text
MLDSA65.Verify(pk, m, sigma) = accept
```

and prove that the distribution of accepted aggregate signatures is
computationally indistinguishable from the distribution required by standard
ML-DSA-65 signing with the same public key and message, subject to the chosen
threshold security model.

## Lemma A: Local Mask Commitment Before Challenge

Statement: Each participant's local mask commitment is fixed before challenge
derivation. The challenge `c` is a deterministic random-oracle output over the
session ID, validator set, public key, message, and ordered commitments.

Why it matters: Participants must not adapt `y_i` after seeing `c`, or the
Fiat-Shamir-with-aborts distribution no longer matches ML-DSA.

Repository status: The simulation transcript binds ordered commitments before
the challenge, but the commitment is a hash fixture rather than a real ML-DSA
mask commitment.

Remaining proof work:

- Define the real commitment value, including whether it commits to `w1_i`,
  `A*y_i`, an opening, or protocol-specific auxiliary material.
- Prove commitment hiding/binding properties required by the selected
  threshold protocol.
- Bind attempt counters and retry IDs so aborted attempts cannot be replayed or
  mixed.

## Lemma B: Aggregate Mask Distribution

Statement: Conditioned on a fixed active set `A`, the aggregate mask:

```text
y = sum_{i in A} lambda_i(A) * y_i
```

has the same distribution, or an explicitly proven acceptable distribution, as
the single-signer ML-DSA mask `y`.

Why it matters: Standard ML-DSA security relies on the distribution of `y` and
on rejection sampling removing challenge-dependent leakage.

Required assumptions:

- Local mask shares are sampled from the protocol-specified distribution.
- The threshold protocol plain-sums the additive mask shares `y_i`, which are
  NOT Lagrange-scaled (`y = sum_{i in A} y_i mod q`); Lagrange weighting
  `lambda_i(A)` is applied only to the Shamir secret share `s1_i` inside each
  `z_i = y_i + c*(lambda_i(A)*s1_i)`. Any residual Lagrange-scaled aggregate
  form written elsewhere in this plan is superseded by the implemented additive
  path (`src/backend/fips_sign.rs` `emit_additive_mask_partial` /
  `aggregate_additive_mask_partials`).
- The aggregate distribution has the correct support and min-entropy.

Remaining proof work:

- Decide whether mask sharing is additive, Shamir-shared, dealerless, or
  generated by another MPC protocol.
- Prove Lagrange recombination does not widen coefficients beyond the planned
  distribution, or add rejection checks that restore the required distribution.
- Quantify Renyi divergence from the standard ML-DSA mask distribution for
  `epsilon_mask`.

## Lemma C: Response Algebra and Norm Bound

Statement: For accepted executions, the aggregate response satisfies:

```text
z = y + c * s
```

and:

```text
||z||_inf < B_z
```

where `B_z` is the standard ML-DSA-65 response bound.

Why it matters: This is the core bound checked by standard ML-DSA verification.

Repository status: `Poly::check_noise_bounds(bound)` can test a single
coefficient array against a caller-supplied bound. The real module-vector bound
checks are not implemented.

Remaining proof work:

- Define centered coefficient conversion for every polynomial in the response.
- Lift scalar `Poly` bounds to ML-DSA module vectors.
- Prove the aggregate check runs after recombination and before signature
  output.
- Prove rejected attempts reveal only allowed abort information.

## Lemma D: Infinity-Norm Bound Preservation

Statement: If the backend accepts a final aggregate only when the actual
encoded aggregate response `z` satisfies all ML-DSA-65 norm predicates, then
aggregation preserves verifier-side infinity-norm validity.

More formally, for final signature `sigma = Encode(c, z, h)`, acceptance by
the threshold backend implies:

```text
forall coefficients a of z: |center(a)| < B_z
```

and all hint and challenge predicates required by ML-DSA-65 hold.

Remaining proof work:

- Specify exact constants and strict-vs-nonstrict inequalities.
- Include hint-count bound `omega` and challenge weight checks.
- Add negative tests for boundary coefficients, malformed hints, and challenge
  mismatch.

## Lemma E: Local Rejection Soundness

Statement: If a participant emits a partial response, then the participant has
verified the local rejection predicate required by the threshold protocol:

```text
LocalAccept(i, transcript, commitment_i, z_i, proof_i) = true.
```

This local predicate must imply that the partial response is consistent with
the committed local mask and the participant's secret share.

Remaining proof work:

- Define `LocalAccept` for the selected threshold ML-DSA protocol.
- Define fraud-proof or slashing evidence for invalid local partials.
- Prove local checks are necessary for liveness/accountability but not
  sufficient for final standard verification unless the aggregate checks also
  pass.

## Lemma F: Aggregate Rejection Soundness

Statement: The aggregator outputs a signature only if the aggregate rejection
predicate accepts:

```text
AggregateAccept(pk, transcript, A, {partial_i}_{i in A}, sigma) = true.
```

This predicate must include:

- Active-set consistency with Lagrange coefficients.
- Challenge consistency with the transcript.
- Partial-share provenance and duplicate rejection.
- Recomputed aggregate response and hint validity.
- Standard ML-DSA verification, or checks proven equivalent to it.

Repository status: `SimulatedAggregator` checks threshold and validator-universe
matching, then delegates to a simulated hash backend. It does not perform real
ML-DSA aggregate rejection checks.

Remaining proof work:

- Implement aggregate recomputation over real ML-DSA-65 values.
- Add standard verifier acceptance as a required backend gate.
- Prove canonical ordering prevents signature malleability from network order.

## Lemma G: Abort Distribution Obligation

Statement: The observable abort behavior of the threshold protocol must not
leak more information about the secret shares than standard ML-DSA aborts leak
about the signing secret. Conditioned on public transcript data, the probability
of accept/abort must match the accepted protocol analysis.

The proof must account for these observables:

- Which participant aborted locally.
- Whether the aggregate attempt aborted.
- Number of attempts before success.
- Timing and message sizes, if visible to adversaries.
- Slashing evidence emitted for invalid partials.

Remaining proof work:

- Define the adversary's view for honest-majority and malicious-minority
  settings.
- Prove local aborts can be simulated from public data, or modify the protocol
  so local aborts are hidden until an aggregate decision.
- Prove retry transcripts are domain-separated and cannot bias future
  challenges.
- Quantify expected retry counts and denial-of-service limits.

## Lemma H: Accepted-Signature Distribution

Statement: The distribution of accepted threshold signatures is the standard
ML-DSA-65 accepted-signature distribution, or is within a stated Renyi
divergence bound that the security proof accepts.

Proof strategy:

1. Show mask generation is equivalent to standard signing before conditioning.
2. Show challenge derivation is equivalent to the standard verifier challenge
   input.
3. Show rejection predicates condition on the same events as standard ML-DSA.
4. Show aggregation is a deterministic algebraic reconstruction of the
   single-signer response.
5. Apply the standard Fiat-Shamir-with-aborts argument to the reconstructed
   signer.

Remaining proof work:

- Select and cite the exact threshold ML-DSA construction being implemented.
- Prove each hybrid step for ML-DSA-65 parameters.
- Obtain external cryptographic review before production use.

## Required Tests and Artifacts

The proof should be backed by at least these artifacts:

- Unit tests for duplicate-free active-set interpolation and malformed active
  sets.
- Property tests for Shamir reconstruction over random `Poly` values and
  active sets.
- Boundary tests for centered infinity-norm checks.
- Real-backend tests that aggregate threshold partials and pass a standard
  ML-DSA-65 verifier.
- Tests that rejected attempts produce no final signature and cannot be reused
  under another transcript.
- Differential tests comparing single-signer and threshold distributions where
  the construction permits empirical sanity checks.

## Exactly What Remains to Be Proven

Before a production claim can be made, the project still needs proofs of:

- Correct field and ring arithmetic for all ML-DSA-65 backend operations.
- Correct Shamir reconstruction for every secret and mask component used by the
  threshold protocol.
- Correct partial-response verification against public commitments.
- Correct aggregate response, challenge, and hint construction.
- Standard ML-DSA verifier compatibility for every emitted aggregate
  signature.
- Preservation of all ML-DSA-65 infinity-norm, hint, and challenge bounds.
- Rejection-sampling distribution equivalence or a quantified Renyi divergence
  bound accepted by the proof.
- Abort leakage bounds for local and aggregate abort behavior.
- Canonical encoding and domain separation for all transcript fields and retry
  attempts.
- Constant-time and side-channel properties of secret-dependent backend code.
- Accountability and slashing soundness for invalid partial shares, without
  introducing additional secret leakage.

Until those items are complete, the current repository should continue to be
described as simulation and research scaffolding only.
